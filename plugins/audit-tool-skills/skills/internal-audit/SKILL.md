# 内审技能（Internal Audit）

## 技能定位

本技能是内部审计能力的总入口与调度中枢。它本身不直接执行具体审计，而是：

1. 理解用户的审计需求；
2. 将需求匹配到对应的模块与审计程序；
3. 调用对应的审计程序文件执行具体审计工作。

## 审计目标

内审围绕以下五项核心目标展开：

- **合规性**：是否符合法律法规与公司制度
- **真实性**：业务与单据是否真实发生
- **准确性**：金额、科目、税率等是否准确
- **完整性**：附件、审批、记录是否完整
- **效益性**：支出是否合理、有无浪费

## 程序目录结构

本技能在 `审计程序/` 文件夹下采用「扁平 + 分级编号」组织，一级文件汇总索引该模块所有二级程序，便于快速定位：

```
internal-audit/
├── SKILL.md                         # 主技能（本文件，整体概括与调度）
└── 审计程序/
    ├── 101-财务模块.md               # ✅ 一级（索引财务模块所有二级程序）
    ├── 10101-财务报销管理.md         # ✅ 二级（已开发）
    ├── 102-人事模块.md               # 一级（规划中）
    ├── 103-采购与供应商模块.md       # 一级（规划中）
    ├── 104-合同管理模块.md           # 一级（规划中）
    └── 105-资产与存货模块.md         # 一级（规划中）
```

### 编号规则

- **一级**（3 位，`101`、`102`……）：业务模块，文件名 `编号-模块名.md`，作为该模块索引页，汇总其下所有二级程序。
- **二级**（一级后接 2 位，`10101`、`10102`……）：具体审计程序，文件名 `编号-程序名.md`，为可执行程序。

新增程序时，在 `审计程序/` 下新建对应 `.md` 文件，并在所属一级文件的索引表中登记；若一级文件不存在则一并新建。

## 通用内审方法论框架

无论哪个程序，均遵循以下通用流程：

1. **计划**：明确审计目标、范围、抽样策略与时间安排
2. **取证**：收集单据、凭证、审批记录、系统数据等证据
3. **分析**：合规、真实、准确、完整、异常多维度比对分析
4. **结论**：形成问题清单与定性判断
5. **报告**：出具审计底稿与报告，跟进整改

### 有限材料下的工作原则（重要）

内审取证常受限于材料完整度。执行时遵循以下原则：

1. **基于现有材料得出有限结论**：只对材料能支撑的判断下结论，不臆测、不外推。
2. **显式标注材料缺口**：每一项结论须注明其依赖的证据是否充分；证据不足时降级为「疑点 / 待核实」，不下定性结论。
3. **主动引导补充信息**：针对缺口，向用户明确列出「还需要补充哪些材料 / 字段 / 制度依据」才能升级结论，并说明补充后能验证什么。
4. **结论分级**：区分「事实（已证实）」「疑点（需补充核实）」「风险提示（模式异常但未定性）」三类输出。

## 调度说明（如何使用本技能）

当用户描述一个内审需求时：

1. 提取需求中的**场景关键词**（如"报销单""发票""差旅""业务招待费"等）；
2. 对照上方程序清单定位所属**一级模块**，打开其索引页（如 `101-财务模块.md`）；
3. 在一级索引页中按关键词匹配到具体**二级程序**，打开并执行其 `.md` 文件；
4. 若一级模块或二级程序未开发（规划中），告知用户该场景暂未落地，可先按"通用内审方法论框架"提供框架性协助。

### 关键词映射示例

- "报销 / 发票 / 差旅 / 业务招待费 / 付款申请 / 网约车 / 住宿费" → 一级 `101-财务模块` → 二级 `10101-财务报销管理`
- "采购 / 供应商 / 招投标 / 入库" → 采购与供应商模块（规划中）
- "合同 / 款项 / 履约" → 合同管理模块（规划中）
- "工资 / 社保 / 考勤 / 离职补偿" → 人事模块（规划中）

## 文件解析说明

用户上传的报销材料（Excel 台账、PDF 发票、行程单、截图等）需先解析为结构化字段后，方可套用审计程序。**文件解析能力（各格式字段提取规则）将另行编写**，本主技能暂不展开。当前阶段：先按用户已提交的解析后数据 / 可读字段执行审计程序。
