# 10101 财务报销管理审计程序

> 属于 [[internal-audit]] 主技能 / [[101-finance-module]] 一级财务模块。本程序为费用报销专项审计的可执行程序文件。

## 一、程序目标与定位

对费用报销业务进行稽核，确认其**合规、真实、准确、完整**，识别违规、虚假、重复或异常报销行为，形成问题清单与定性结论。

### 审计策略定位（先认清投入产出）

- **容易出结果，但整体 ROI 偏低**：单案舞弊金额通常不高，台账整理需耗费大量人力时间。
- **适用场景**：更适合作为**被动型专项审计**（收到线索后针对性核查），不建议作为年度常规审计的核心主攻板块。
- **附加价值**：具备较强的全员行为震慑作用，可作为规范报销管理的工具。

> 执行前先与用户确认本次审计是"线索驱动的专项核查"还是"常规抽查"，以决定投入深度。

## 二、审计基础：全维度台账底稿搭建

这是所有费用报销审计的前提。通过标准化字段归集信息，为后续交叉验证提供数据基础。

需采集的核心字段：

| 类别 | 字段 |
| --- | --- |
| 报销基础信息 | 报销单号、报销金额、费用描述、报销日期 |
| 出差审批信息 | 出差申请单号、申请日期、出差事由、出差地点 |
| 人员与主体信息 | 报销人、开票单位、收款方、发票号码 |
| 行程明细信息 | 机票 / 高铁 / 打车 / 住宿的时间、地点；网约车乘车人手机号 |
| 主体风险辅助信息 | 报销人常驻地、报销人户籍 / 老家所在地、开票单位经营状态 |

> **材料缺口提示**：若用户提交材料缺少上述任一类别，台账不完整，后续多项交叉校验无法执行。需在底稿中标注"字段缺失"，并按第六节引导补充。

## 三、基础合规性核查

### 1. 出差审批匹配性

- 核查实际报销的出差时间、地点与出差申请单是否一致。
- 排查**无出差申请、事后补申请即报销**的违规情形（重点看申请日期晚于出差开始日期 / 报销日期）。

### 2. 制度标准符合性

- 对照公司报销制度，逐类校验差旅、住宿、招待等费用是否超报销标准。
- 重点关注**「贴额度报销」行为**：每月报销金额持续接近、顶格使用报销额度。

> **材料缺口提示**：制度标准符合性必须依赖公司《报销管理制度》《费用标准表》《审批权限表》。若用户未提供，本项只能做"贴额度"模式识别，无法做"超标准"判定。

## 四、核心真实性核查（舞弊识别核心）

### （一）通用逻辑：时间地点交叉校验

这是最高效的舞弊识别方法，出问题概率极高：

1. 提取所有带时间、地点属性的票据（机票、高铁、打车、酒店）信息。
2. **校验同一报销人是否存在「同一时间、不同地点」的报销记录**——直接识别重复报销、虚构行程。
3. 校验整体行程轨迹是否与报销事由（客户拜访、办公等）逻辑匹配。

> 此项只要台账中时间、地点字段齐全即可执行，是有限材料下性价比最高的核查。

### （二）场景化突破点 1：有商旅系统、打车 / 招待垫付、无需支付记录

这类场景下，**打车 / 网约车费用是最高效的审计突破口**：

1. **乘车身份核验**：网约车行程单显示的手机号是否为报销员工本人号码，非本人号码为典型疑点。
2. **时间冲突排查**：同日内是否存在时间重叠、地点矛盾的多笔打车记录。
3. **行程合理性**：起止地点、打车时段与报销事由是否匹配（如事由为商务拜访，但行程集中在居民区、凌晨高频打车等异常）。

### （三）场景化突破点 2：无商旅系统、全员垫付、要求提交支付记录

这类场景下，**住宿费、餐饮招待费是核心核查方向**：

1. **发票真伪校验**
   - 通过税务官方平台核验，排查"查无此票""票面金额与税务系统登记金额不符"的假票。
   - 核查开票主体经营状态：开票日期对应商家已被吊销 / 注销的，直接指向虚开发票。
2. **支付记录真实性**
   - 排查是否大量以个人转账（而非对公收款、正规刷卡）作为支付凭证。
   - 识别支付截图 P 图痕迹、故意遮挡收款方信息的异常。
   - 关注资金闭环风险：是否存在付款后资金原路转回的虚假支付。
3. **额度异常特征**：每月报销金额持续紧贴公司额度上限。

### （四）场景化突破点 3：商旅平台合作订酒店 —— 员工与酒店串通返差价

> 典型舞弊场景（来自实务案例）：公司与商旅平台合作统一订酒店，员工与特定酒店串通，以高于市场的价格开票结算，差价由酒店私下返还给员工。

识别特征（多个特征叠加方可提高置信度，单一特征不构成定性）：

1. **费用持续贴额度**：该员工酒店费用长期接近报销标准上限。
2. **酒店价格同比偏高**：同城市、同档次、同时段，该员工入住酒店价格明显高于市场公开价 / 平台均价 / 同事入住价。
3. **固定少数几家酒店**：出差地分散，但每次都集中在固定的几家酒店消费，缺乏正常出差的酒店随机性。
4. **开票主体异常**：开票酒店与出差地一致但选择面极窄；或开票主体为个体户 / 小规模纳税人且与平台挂牌主体不一致。

> **材料缺口提示**：本场景需"市场公允价"作对照（同城市同档次酒店均价）。若仅有公司内部报销数据、无外部比价数据，只能识别"固定酒店 + 贴额度"的结构性疑点，无法直接证明"价格偏高"，需补充市场价数据或同期间同事入住记录。

## 五、发票与主体风险排查

### 1. 发票真实性核验

无自动验真系统时，需通过官方税务渠道人工核验发票真伪。

### 2. 替票风险信号

- **开票地与出差地严重不符**：开票单位所在地与报销事由的出差地严重不符（如员工常驻广东、事由为外省出差，但大量发票来自本地），是典型的替票 / 虚假报销信号。

- **📌 户籍 / 老家发票聚集（私费公报控制点）**：
  - 调取报销人**户籍所在地或老家所在地**信息。
  - 统计开票单位落在上述地点的发票**比例与金额**。
  - 若该比例显著偏高（如出差事由与该地无关，却反复出现该地餐饮、住宿、购物发票），大概率是**私费公报**——员工将个人回乡 / 探亲 / 日常消费开票后回公司报销。
  - 判定时结合出差事由：若出差地确为户籍地则属正常，需区分"合理出差回乡"与"无出差事由的老家发票聚集"。

### 3. 印章真实性

通过公安 / 官方备案渠道核查发票、附件的公章真实性，排查假公章。

## 六、深度延伸验证手段

当上述核查形成完整疑点链后，按以下顺序闭环：

1. **商家反向核实**：针对大额、高疑点的餐饮 / 住宿发票，直接联系开票商家，核实消费记录、开票记录、开票人员身份真实性。
2. **当事人面谈核实**：锁定完整疑点证据后，与报销人当面核实，确认虚假报销事实。
3. **闭环处理**：核实舞弊后，追回违规报销款项，按公司制度做辞退、通报等处理。

> 延伸验证手段涉及对外联系与人事动作，须在内部疑点证据充分后再启动，避免误伤。

## 七、有限材料下的执行规则（重要）

本程序在材料不完整时仍可启动，但必须遵守以下规则：

### 1. 结论分级

| 级别 | 含义 | 输出要求 |
| --- | --- | --- |
| **事实** | 材料已证实 | 给出定性结论 + 证据索引 |
| **疑点** | 模式异常但证据不足 | 列明异常点 + 所缺证据，不下定性 |
| **风险提示** | 统计层面偏离常态 | 提示关注，建议持续监控 |

### 2. 材料缺口 → 结论降级

- 缺出差申请单 → "事后补申请"核查降级为疑点。
- 缺制度标准 → "超标准"核查不可执行，仅保留"贴额度"模式识别。
- 缺支付记录 → 场景 2 的资金闭环核查降级为疑点。
- 缺市场公允价 → 场景 3 的"价格偏高"不可定性，仅保留"固定酒店 + 贴额度"结构疑点。
- 缺户籍 / 老家信息 → 私费公报控制点不可执行。
- 缺网约车手机号 → 场景 1 的乘车身份核验不可执行。

### 3. 引导补充信息模板

每次输出结论时，附"补充材料清单"，逐项说明：

- **需要补充什么**（具体字段 / 文件 / 制度）；
- **补充后能验证什么**（对应升级哪一项结论）；
- **当前不补充的影响**（哪项核查只能停留在疑点级）。

## 八、输出模板

审计产出应包含：

1. **全维度台账底稿**：按第二节字段归集的结构化表。
2. **问题清单**：问题编号、报销单号、报销人、问题类型、风险等级、结论分级（事实 / 疑点 / 风险提示）、证据索引、建议。
3. **补充材料清单**：按第七节模板列出待补充项及预期验证效果。
4. **审计小结**：总体结论、覆盖范围说明（哪些维度已核查、哪些因材料缺失未核查）、风险提示、整改建议。

## 九、重要声明

- 本程序提供**可执行的审计步骤与核查逻辑**，但不含具体金额标准与制度细则——需用户补充公司制度依据方可做"超标准"类实质性判定。
- 审计结论应基于充分证据，**避免仅凭单点异常下定性结论**；多特征叠加方可提高置信度。
- 涉及对外核实、人事处理等动作，须在内部证据链完整后启动。
